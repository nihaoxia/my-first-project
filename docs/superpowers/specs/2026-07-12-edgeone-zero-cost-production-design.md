## 1. 背景与约束

Stray Pages 当前代码已经具备基于腾讯云广州 CVM、Docker Compose、自托管 Supabase/PostgreSQL、腾讯云 COS、腾讯云短信 Hook、Translation MCP 和 TCR 的生产部署实现与运行手册。该架构通过了现有测试、构建和 CI，但依赖固定费用或按量计费资源，不能满足新的最高优先级约束。

本设计以以下约束为不可放宽的硬边界：

1. 不购买或启用任何固定费用、按量计费或到期自动转收费的资源。
2. 不使用 CVM、轻量应用服务器、收费域名、腾讯云短信、COS 生产读写、TCR 企业版或收费模型 API。
3. 不把限时试用、首月优惠、代金券或可自动超额计费的额度视为永久免费。
4. 不把账号密码、SecretId、SecretKey、OTP、恢复码或其他敏感值写入聊天、Git、构建日志或应用日志。
5. 生产能力必须在免费额度耗尽前硬停止，禁止自动降级到任何收费服务。
6. 服务商选择保持国内优先。生产运行平台使用腾讯 EdgeOne Makers；GitHub 继续作为用户指定的源代码仓库，但部署不强制依赖 GitHub 在线构建，可使用本地 CLI 或直接上传流程。
7. 不宣称现有腾讯云服务器架构已正式上线；只有零费用架构完成部署和验收后，才能标记生产任务完成。

## 3. 方案比较与选择

### 3.1 方案一：EdgeOne Makers 永久免费版（采用）

使用 EdgeOne Makers 托管 Next.js，使用 Makers Functions 运行服务端 API，使用具备强一致读取和条件创建能力的 Blob 保存权威账号、结构化业务数据、原文、章节、译文与导出文件。KV 仅保存允许短暂陈旧且可重建的缓存。使用平台免费域名，不购买自定义域名。

优点：腾讯产品、支持中国访问、无需常驻服务器、免费版官方声明永久提供、具备 Next.js 全栈与存储能力。缺点：不能原样运行 Docker、PostgreSQL 或 Supabase；Blob 不是关系数据库，需要改写认证、数据访问、索引和并发控制；免费存储和函数额度有限。

### 3.2 方案二：用户本机 Docker 自托管（保留为开发与离线退路）

继续在用户电脑运行现有 Docker/PostgreSQL 或本地模式，只提供本机或局域网访问。

优点：不产生云资源费用，可保留现有关系数据库架构。缺点：电脑必须开机，不能视为稳定公网生产服务，公网穿透、域名和长期可用性都无法在绝对零费用前提下保证。

### 3.3 方案三：限时免费云服务器或 CloudBase（拒绝）

此方案能减少代码迁移量，但免费期有限，到期后付费，违反硬约束。

## 5. 账号与认证设计

### 5.1 登录方式

生产账号使用 Stray Pages 自有的用户名和密码，不依赖短信、收费邮件或国外身份提供商。这里的登录仅用于隔离 Stray Pages 云端数据，不要求最终用户登录 GitHub、Supabase 或其他第三方服务。

注册流程：

1. 用户提交用户名和密码。
2. 服务端规范化用户名，验证长度和允许字符。
3. 服务端生成随机用户 ID。
4. 密码使用内存困难型密码哈希算法和每用户随机盐处理；Blob 中只保存哈希参数、盐和结果。
5. 服务端生成一次性恢复码，仅在注册成功响应中展示一次；Blob 只保存恢复码哈希。
6. 创建短期登录 Session，并通过 `HttpOnly`、`Secure`、`SameSite=Lax` Cookie 返回。

### 5.2 会话

- Cookie 仅包含高熵随机 Session Token，不包含用户资料或权限声明。
- Blob 中以 Token 的密码学哈希作为对象键，保存用户 ID、创建时间、最后活动时间和过期时间；读取使用强一致模式。
- 每次受保护请求都在服务端强一致读取并验证 Session、过期时间和用户状态。
- 退出时删除服务端 Session；客户端 Cookie 随即失效。
- Session 有绝对有效期和空闲有效期，不能无限续期。
- 修改密码或使用恢复码后撤销该用户全部 Session。

### 5.3 密码恢复

- 不发送短信或邮件。
- 用户输入用户名、一次性恢复码和新密码。
- 恢复成功后轮换恢复码并撤销全部 Session。
- 恢复码仅展示一次；丢失恢复码且忘记密码时，系统不能以牺牲账号安全为代价提供人工绕过。

### 5.4 防护

- 注册、登录和恢复接口按客户端与用户名双维度限速。
- 错误响应不能区分“用户名不存在”和“密码错误”。
- 日志不得包含用户名原值、密码、Session Token、恢复码、Cookie 或上传正文。
- 所有用户资源操作必须从服务端 Session 推导用户 ID；不得信任客户端提交的用户 ID。

## 6. 权威数据、一致性与索引

### 6.1 平台一致性边界

EdgeOne KV 是多边缘节点最终一致存储，其他节点最长可能在 60 秒内读到旧值，且公开 API 只有普通覆盖写，不提供基于版本或 ETag 的条件更新。因此 KV 不得保存或裁决以下安全和正确性关键数据：密码、Session、恢复码、唯一用户名、权限、用量余额、资源所有权或当前业务版本。

EdgeOne Blob 支持强一致读取、强一致列举和 `onlyIfNew` 条件创建。本设计使用 Blob 作为唯一权威存储；KV 只缓存可由 Blob 重建、允许短暂陈旧且不会改变授权或计费判断的数据。

权威参考：

- <https://pages.edgeone.ai/zh/document/kv-storage>
- <https://pages.edgeone.ai/zh/document/blob-storage>

### 6.2 Blob 键空间

Blob 使用稳定键空间。核心结构如下：

```text
auth/accounts/{usernameHash}/claim.json
auth/accounts/{usernameHash}/revisions/{revisionId}.json
auth/sessions/{sessionHash}.json

data/{userId}/{resourceType}/{resourceId}/revisions/{revisionId}.json
data/{userId}/{resourceType}/{resourceId}/content/{contentId}.txt
indexes/{userId}/{resourceType}/events/{eventId}.json
usage/{userId}/{yyyy-mm}/events/{eventId}.json
uploads/{userId}/{reservationId}.json
```

设计要求：

- 所有业务键都包含从 Session 得到的用户 ID。
- 用户名索引使用规范化用户名的加密哈希，不在键名中暴露原始用户名。
- 用户名声明、账号修订、Session、业务修订、索引事件和用量事件均使用高熵 ID 或哈希键，并通过 `onlyIfNew` 创建，禁止覆盖已有安全记录。
- 每次业务修改写入新的不可变 Revision。Revision 包含 `parentRevisionId`、`operationId`、`createdAt`、内容引用和业务元数据。
- 读取资源时强一致列举 Revision，并按父子关系求出当前叶节点。如果同一父 Revision 出现多个子 Revision，则返回 `VERSION_CONFLICT`，保留所有分支并等待显式解决，不能静默采用最后写入者。
- 冲突解决创建一个引用所有冲突叶节点的新合并 Revision；旧版本不原地修改。
- 列表索引使用不可变增删事件。权威列表通过强一致列举事件重建；KV 可以缓存结果，但授权判断和修改前检查不得依赖该缓存。
- 多对象工作流使用高熵 `operationId` 和不可变步骤记录保证幂等。任何中间失败都能从已有步骤继续，不覆盖已完成步骤。
- 删除以不可变删除 Revision 表示。物理 Blob 清理在删除状态确认后幂等执行；清理失败不会让对象重新可见。
- 正文和译文与元数据使用不同对象，避免读取列表时下载大文本。

### 6.3 用户名、Session 与恢复码原子性

- 注册先生成账号根 Revision，再以 `onlyIfNew` 创建 `auth/accounts/{usernameHash}/claim.json`，声明对象引用用户 ID 和根 Revision；声明已存在时注册失败，从而保证用户名唯一。根 Revision 写入成功但声明失败时只是不可寻址孤儿，可由对账流程清理，不影响其他账号。
- 密码哈希、恢复码哈希和账号级会话世代保存在不可变账号 Revision 中。修改密码或使用恢复码时，以当前账号 Revision 为父节点创建新 Revision；出现并发分支时暂停该账号的新登录和恢复操作，直到安全解决冲突。
- Session 和恢复码使用不可预测随机值；Blob 只保存 Session Token 哈希和恢复码哈希。Session 对象引用创建时的账号会话世代；每次请求强一致读取账号 Revision，世代不匹配即拒绝，从而实现全局撤销。
- KV 中不得保存可被直接用于认证的 Token、密码哈希或恢复码哈希。

## 7. Blob 对象存储设计

文件对象键由服务端生成：

```text
users/{userId}/books/{bookId}/original.txt
users/{userId}/books/{bookId}/chapters/{chapterId}.txt
users/{userId}/translations/{translationId}/{chapterId}.txt
users/{userId}/exports/{exportId}.txt
```

安全和配额要求：

- Blob 不允许匿名列举或任意对象键访问。
- 上传前验证 TXT 类型、UTF-8、现有产品文件大小上限和用户剩余配额。
- 平台单对象上限为 25 MB；应用上限必须不高于该限制。
- 下载和删除先验证 Session、元数据所有权与对象键前缀。
- 用户上传总量通过 Blob 中的不可变预留和结算事件计量；对失败重试使用幂等操作 ID，避免重复扣减或重复释放。
- 整个平台 Blob 免费配额为 1 GB。系统设置低于平台上限的安全水位，达到安全水位后拒绝新上传，不申请或启用收费扩容。
- 生产代码中不得存在 COS 自动回退、COS 双写或超额后转 COS 的逻辑。
- 客户端直传前先按“单次允许上传的完整应用上限”而不是客户端声明大小创建不可变预留事件，再签发短期、固定对象键和固定 `Content-Type` 的 PUT URL。所有未完成预留都计入总量，防止并发签发导致超配额。上传完成后以强一致读取验证对象实际大小并写入结算事件，释放预留与实际大小的差额；过期预留由对账流程释放。应用安全水位还必须预留至少一个平台最大对象的余量，防止客户端声明大小不可信时突破 1 GB 平台上限。

## 8. 免费额度与硬停止

官方当前免费配额包括：40 个项目、500 次构建/月、5 GB 项目存储、300 万次 Edge Functions/月、100 万次 Cloud Functions/月、50 万 GB-s/月 Cloud Functions、1 GB KV 和 1 GB Blob。

应用必须实施双层保护：

1. **应用内保守账本**：使用 Blob 不可变事件跟踪用户存储量、预留量、上传次数、模型 Token 和高成本操作，使用低于平台配额的阈值提前停止。KV 只能缓存计算结果，不能作为是否允许继续消耗额度的权威判断。
2. **部署前人工门禁**：每次生产发布前核对 EdgeOne 官方价格与配额页面，确认仍为永久免费、无需信用卡且没有自动超额计费。

当账本不可用、配额状态不明或平台政策无法确认时，写操作 fail closed；允许用户读取和导出已有数据，但不允许产生新的资源用量。

禁止实现以下行为：

- 自动购买、自动续费或自动升级套餐。
- 自动申请付费扩容。
- 超额后调用 COS、短信、收费数据库或收费模型。
- 以代金券余额或试用余额作为可用生产预算。

## 9. 翻译能力

零费用生产模式保留本地翻译、手动导入译文和可验证的免费模型路径。腾讯混元、OpenAI 或任何按 Token 计费 API 不得成为默认生产依赖。

EdgeOne Makers 当前提供每月 50 万内置模型免费 Token 时，可在满足以下条件后接入：

- 官方确认调用不会自动产生费用。
- 使用服务端月度 Token 账本和低于 50 万的硬停止阈值。
- 每个任务在执行前计算保守上限并预留额度。
- 额度不足时任务拒绝启动，不切换到收费 Provider。
- 界面明确说明免费额度有限，不能承诺无限量整本翻译。

Translation MCP 保留为本地开发和可选集成边界，不部署常驻容器服务。若 EdgeOne Functions 无法在免费配额内安全承载某项 MCP 功能，该功能仅在本地模式开放。

## 13. 错误处理与可观测性

- 对外错误使用稳定的应用错误码，不返回 EdgeOne、KV、Blob 或模型供应商原始错误。
- 配额不足统一返回可识别的 `FREE_QUOTA_EXHAUSTED`，界面说明数据仍可读取和导出，但新写入已暂停。
- 并发冲突返回 `VERSION_CONFLICT`，客户端重新读取后由用户决定是否重试。
- 存储账本不一致时返回 `USAGE_LEDGER_UNAVAILABLE` 并停止写入。
- 认证失败不暴露账号是否存在。
- 日志只记录请求 ID、稳定错误码、耗时和脱敏资源类型，不记录正文、译文、用户名原值、密码、恢复码、Session、Cookie 或平台凭据。

## 14. 测试与验收

### 14.1 单元与契约测试

- 用户名规范化、密码哈希参数、恢复码哈希和 Session 生命周期。
- 登录与恢复接口限速及防账号枚举。
- Blob 键空间、强一致读取、条件创建、不可变版本、冲突检测、幂等重试和索引修复。
- KV 缓存不参与认证、授权、配额和当前版本裁决。
- Blob 对象键、所有权校验、文件策略和用量账本。
- 免费额度安全水位与所有收费 Provider 的 fail closed 行为。
- 生产配置禁止 COS、短信、CVM、付费数据库、收费模型和付费域名。
- 旧本地模式继续通过既有测试。

### 14.2 集成验收

1. 两个账号注册、登录、退出与恢复码轮换成功。
2. 两个账号的书籍、译文、笔记和 Blob 对象互不可见、不可修改、不可删除。
3. 换浏览器 Session 后，用户能读取自己的云端数据。
4. 并发修改同一资源时不会静默覆盖。
5. 上传、下载、删除和失败重试不会造成用量账本漂移。
6. 达到应用安全水位后写入和模型调用硬停止，读取与导出仍可用。
7. 不配置 COS、短信、数据库或收费模型凭据时生产构建和运行正常。
8. EdgeOne 免费域名从中国网络环境可访问。
9. GitHub CI 全部通过，生产部署固定到已提交 SHA。

### 14.3 费用验收

- 腾讯云账号中不存在 CVM、轻量服务器、TCR 企业实例或短信发送记录。
- COS Bucket 没有生产对象和生产请求路径。
- EdgeOne 开通页面显示免费版、无需信用卡，没有自动续费或自动超额付费。

# 同账号本地学习数据安全导入云端设计

## 背景与审计结论

仓库已经具备本地学习数据导入云端的服务、API、客户端分块、幂等回执和用户入口。现有入口位于云端笔记页，会一次读取当前账号作用域、一个名为“旧 Mock”的作用域和历史未分区固定键，然后使用原生确认框直接开始导入。

审计确认以下边界需要收口：

- 所谓“旧 Mock”作用域由当前 Session 用户 ID 再次派生，和 `AppShell` 提供的当前账号作用域相同，并不是一个可证明不同的历史账号来源；继续展示该说法会夸大迁移能力。
- 当前流程没有独立的零网络检查、来源选择、分类选择和数量预览，用户只能接受或取消一次汇总确认。
- 版本 1 完成标记会永久阻止再次检查，导致首次导入后新增的本地记录无法继续通过同一入口导入；服务端本来已经使用持久化回执保证重复执行幂等，不需要客户端永久封锁。
- 入口藏在笔记页，不符合“我的”页已经承担账号、本地备份与恢复管理的产品结构。
- 现有导入只覆盖词汇、句子和自由笔记；缺少来源关系的阅读器纯文本收藏会被计为无法映射。原书、译本、自动同步、云端自动备份和跨账号迁移不在现有协议能力内。

## 目标

在不新增依赖、不创建平台资源、不绕过零费用写门禁的前提下，把现有同账号本地学习数据导入整理为一个可发现、可选择、可预览、可安全重试的完整用户流程：

1. 在云端持久化模式且当前 Session 可用时，于“我的”页展示数据迁移入口。
2. 检查阶段只读取固定本地键，不发起网络请求，不写入本地或云端数据。
3. 明确区分当前账号作用域与历史未分区数据；历史未分区来源默认不选中。
4. 允许按来源和词汇、句子、笔记三类数据选择导入范围。
5. 在执行前展示只含数量的预览，并要求显式确认。
6. 预览后本地源数据发生变化时，在首次网络请求前拒绝执行并要求重新检查。
7. 保留本地副本，依靠现有服务端回执实现安全重试；完成标记只记录最近一次完整结果，不再阻止后续检查。
8. 保持账号所有权、正文校验、请求预算、分块、冲突统计、Blob 写门禁和项目额度门禁全部由现有服务端权威执行。

## 非目标

- 不迁移本地原书、译本、翻译任务、阅读进度或阅读设置。
- 不实现后台自动同步、定时备份、跨设备实时同步或跨账号迁移。
- 不根据标题自动创建缺失的云端原书，也不伪造缺少书籍/章节来源的收藏关系。
- 不枚举 `localStorage`，不读取上传草稿、原书、译本、备份文件或其他未列入白名单的键。
- 不上传来源选择、预览对象、原始存储快照或完成标记；只有用户确认选中的合法学习记录会进入既有导入清单。
- 不调用真实 EdgeOne、Blob、KV、Models、COS 或其他云资源完成本轮开发与验证。真实写入继续要求 `EDGEONE_FREE_BLOB_CONFIRMED=true`，当前保持 `false`。

## 方案比较

### 方案 A：增强现有学习数据导入（采用）

保留服务端版本 1 导入协议和幂等回执，重构客户端为检查、选择、预览、确认、执行五个清晰状态；入口迁移到“我的”页。

优点是复用已验证的所有权、分块、回执和冲突模型，不扩大云端数据协议；风险集中在客户端状态失效和选择过滤，可以用纯逻辑测试和 UI 合约测试覆盖。

### 方案 B：直接扩展原书与译本迁移

为结构化章节、对象内容和已完成译本新增导入协议、持久化回执及 Revision 创建路径。

这能更接近完整书架迁移，但原书对象、章节元数据、译本 Revision、任务历史和幂等提交需要独立设计。把它塞进当前里程碑会让一个安全收口任务跨越多个权威存储子系统，不符合可独立验证的小步交付原则。

### 方案 C：把本地章节拼成 TXT 后循环调用普通上传接口

客户端可以重用 `/api/cloud/books`，但服务端会重新按标题行拆章。正文中合法出现的章节式标题可能造成二次切分，进而改变章节边界或内容；逐书请求也缺少跨重试的本地来源回执。该方案不采用。

## 权威来源与固定键

客户端只允许读取以下四个基础键：

- `stray-pages.study-vocabulary`
- `stray-pages.study-sentences`
- `stray-pages.study-notes`
- `stray-pages.reader-selections`

每个基础键只能从以下两个来源读取：

1. `current-supabase-scope`：名称作为既有 `sourceId` 哈希输入保留，以维持已经产生的服务端导入回执兼容性；用户界面统一显示为“当前账号本地数据”。读取通过 `readScopedLocalStorage`，其实际作用域由当前已认证 `AppShell` 的 `data-local-storage-scope` 决定。
2. `legacy-unscoped`：用户界面显示为“历史未分区数据”。读取通过 `readLegacyLocalStorage(storage, key, null)`，只允许安全存储模块中的固定白名单，不允许直接访问任意键。

删除 `legacy-mock-scope:<scope>` 重复来源。应用没有可信的旧账号标识映射时，不声称可以读取另一个旧账号的作用域数据。

## 核心数据模型

在 `import-client-core.ts` 中增加面向预览的纯逻辑类型：

- `LocalStudyImportKind`：`vocabulary | sentence | note`。
- `LocalStudyImportSelection`：固定来源 ID 数组和固定分类数组；未知、重复或空选择均拒绝。
- `LocalStudyImportSourcePreview`：来源 ID、三类可导入数量、无法映射数量和本地非法数量；不含正文。
- `LocalStudyImportPreview`：选中后的总数量、分来源数量、无法映射数量和本地非法数量；不含正文、存储键或快照值。

`buildLocalStudyImportManifest` 继续负责规范化、去重、稳定 `sourceId`、字段预算和清单构建，但接收显式选择：

- 只处理选中的来源和分类。
- 当前账号来源固定排在历史未分区来源之前；同种记录同 ID 时当前账号版本优先。
- 阅读器纯文本收藏没有书籍与章节关系，继续只计入 `unresolved`，不生成伪造记录。
- 任何超过字段预算的记录计入 `localErrors` 和 `unresolved`，合法记录仍可进入预览。
- 清单项、分块及服务端响应验证规则保持不变。

## 用户流程与状态

### 1. 可见性

“我的”页在 `resolveCloudPersistenceMode(getCloudServerConfig()) === "cloud"` 时渲染 `CloudLocalImportPanel`。本地模式不展示云端迁移控件；配置异常时不把本地模式伪装成云端模式。云端笔记页移除重复入口。

### 2. 检查

用户点击“检查本地学习数据”后：

1. 读取两个来源的四个固定键。
2. 任一读取失败或任一固定键内容无法通过权威解析器校验时，整体检查失败，不展示部分可执行候选。
3. 把成功读取的原始字符串快照与解析后来源只保存在当前 React 内存状态中，不写入浏览器存储、不打印日志。
4. 展示两个来源的词汇、句子、笔记和无法映射数量。
5. 默认只选中有数据的当前账号来源；历史未分区来源始终默认不选中。三类可导入分类默认全选。

### 3. 选择与预览

来源或分类选择变化时，立即清除旧预览和确认状态。用户点击“生成导入预览”后，客户端使用选中的来源与分类构造候选清单，并只展示：

- 将尝试导入的词汇、句子和笔记总数；
- 每个选中来源的可导入数量；
- 无法映射和本地非法记录数量；
- 预计分块数量。

没有选中来源、没有选中分类或最终没有任何合法清单项时禁止执行。无法映射或本地非法数量大于零时允许导入其余合法记录，但完成后不得写“完整导入”标记，界面必须明确说明有记录未进入云端。

### 4. 确认与执行

执行按钮前提供独立复选框，确认文案明确说明：只导入预览中的三类记录、本地副本不会删除、缺少唯一云端书籍/章节匹配的记录可能被服务端报告失败。

点击执行后，在任何 `fetch` 之前重新读取预览所涉及来源的四个固定键，并与内存原始快照逐项严格比较：

- 任一读取失败或值发生变化，返回 `SOURCE_DATA_CHANGED`，清除预览与确认，零网络、零写入。
- 当前账号作用域在预览后变化时同样拒绝执行，不能把上一个账号的候选发送到新 Session。
- 快照一致时才调用现有 `runImportChunks`；服务端 Session 再次决定真实用户 ID，客户端不得提交用户 ID。

执行期间锁定检查、来源、分类、确认和执行控件。成功、冲突、服务错误、网络错误和无效响应都使用页面内 `role=status` 或 `role=alert` 文案，不使用 `window.confirm` 或阻塞式弹窗。

### 5. 完成标记与重试

删除“发现版本 1 标记就永久返回”的逻辑。完成标记仍写入当前账号作用域，只用于记录最近一次完整导入结果：

- 所有分块完成；
- `conflicts === 0`；
- `errors === 0`；
- `unresolved === 0`；
- `localErrors === 0`。

标记不包含正文、选择、预览或原始快照。再次检查和导入始终允许；服务端以 `kind + sourceId + payloadHash` 的持久化回执返回 `skipped`，有内容变化的同来源记录返回 `conflict`，不会静默覆盖。

无网络错误或服务端前置拒绝时保留预览，允许用户安全重试；成功或本地源变化时清除预览。任何路径都不删除或修改四类本地源数据。

## 错误与文案边界

- 存储不可用：提示检查浏览器隐私设置，零网络。
- 当前账号作用域不可用：提示重新登录，零网络。
- 固定来源内容损坏：指出对应来源和分类，不展示正文，零网络。
- 空选择：要求至少选择一个来源和一种数据分类。
- 无合法记录：展示无法映射/非法数量，不发送空清单。
- 本地数据变化：提示重新检查，零网络。
- `AUTH_REQUIRED`：提示登录已失效。
- `CLOUD_NOT_CONFIGURED`、`CLOUD_CONFIG_INVALID` 或写门禁拒绝：提示云端写入尚未通过零费用配置，不建议切换收费方案。
- 分块失败：展示失败批次序号、已完成批次数和可安全重试说明，不展示请求正文、清单 ID 或服务端内部路径。
- 部分结果：展示新增、已存在、冲突、失败和未映射数量；不得称为完整同步。

## 安全、隐私与零费用约束

- 客户端不枚举存储，不读取白名单外键，不记录正文或原始值。
- 检查、选择和预览阶段没有 `fetch`，没有本地写入，也没有云端调用。
- 原始快照、选择和预览只存在于内存，不进入导入请求；请求只包含选中的规范化业务记录。
- 路由继续只接受活动 Session，`userId` 只由服务端 Session 注入。
- 服务端继续执行 2 MiB 请求体上限、1 MiB 规范化清单预算、单清单 1,000 项上限、严格字段集合和重复 JSON 键拒绝。
- EdgeOne Blob 写门禁、999 MiB 项目额度和所有权校验不变；`EDGEONE_FREE_BLOB_CONFIRMED=false` 时必须在 SDK 写调用之前拒绝。
- 本轮测试只使用纯逻辑、路由依赖注入、本地开发浏览器和静态扫描；不创建、写入或调用真实云资源。

## 文件边界

- `src/lib/cloud/import-client-core.ts`：选择校验、只含数量的预览、选中过滤和既有清单/分块逻辑。
- `src/components/cloud/cloud-local-import-panel.tsx`：固定来源读取、内存快照、状态失效、可访问控件、执行前复核和结果文案。
- `src/app/me/page.tsx`：只在权威云端持久化模式挂载迁移面板。
- `src/app/study/notes/page.tsx`：移除隐藏的重复入口。
- `tests/cloud-import-client.test.ts`：选择、优先级、预览、空/未知选择和分类过滤的纯逻辑测试。
- `tests/cloud-study-ui-contract.test.ts`：入口位置、固定键、零枚举、零弹窗、快照复核、状态失效和本地副本保留合同。
- `README.md`、`docs/ROADMAP.md`、`docs/DEV_LOG.md`：同步已实现范围与仍未实现范围，并修正“部署已验证 Git SHA”的待办歧义。

## 测试与验收

### 自动化

- TDD 聚焦测试覆盖来源与分类选择、当前来源优先、历史来源显式选择、无法映射统计、非法记录统计、分块预算和响应校验。
- UI 合约测试确认入口位于“我的”页且只在云端模式渲染；笔记页不再挂载入口。
- UI 合约测试确认只使用两个固定来源、四个白名单键，不出现 `localStorage.length`、`localStorage.key()`、`window.confirm`、源数据删除或正文日志。
- UI 合约测试确认选择变化清除预览和确认，执行前复核原始快照，复核通过后才出现 `fetch`。
- 完整运行 `pnpm test`、`pnpm lint`、`pnpm typecheck`、`pnpm build`、`pnpm verify:zero-cost` 和 `git diff --check`。

### 浏览器

- 云端模式下“我的”页可发现入口；本地模式不展示。
- 默认只选当前账号来源，历史未分区来源需要主动勾选。
- 检查与预览期间网络面板没有导入请求。
- 来源/分类变化会清除预览和确认；空选择不能执行。
- 预览后修改一个固定源键，执行时零网络拒绝并要求重新检查。
- 合法请求成功时显示新增/已存在/冲突/失败/未映射数量，本地源键保持原值。
- 桌面与可实现的窄屏视口无横向溢出，键盘可操作，状态文案由辅助技术感知，控制台无产品错误或警告。

## 后续里程碑

本里程碑完成后，单独设计“同账号本地书架手动迁移云端”：它必须为结构化原书章节、对象内容、译本 Revision 和持久化导入回执定义权威协议，不能通过普通 TXT 二次拆章冒充无损迁移。自动同步、云端自动备份与跨账号迁移继续保持未实现。

# 浏览器本地数据加密备份与恢复设计

## 背景

Stray Pages 已把本地原书、译本、学习资料和阅读器收藏保存在浏览器当前账号作用域中。现有存储模块会拒绝覆盖损坏的数据，也不会把本地数据伪装成云端数据；但用户仍可能因为清理浏览器数据、使用无痕模式、设备损坏或更换浏览器而失去全部本地内容。目前项目没有可验证的本地备份与恢复入口。

本阶段增加一个完全在浏览器中运行的加密备份包。用户在“我的”页创建一个文件，并可在同一账号作用域中检查后整体恢复。正文、学习资料和口令不离开当前设备；不使用 EdgeOne、Blob、KV、Models、COS、第三方备份服务或付费资源，也不新增加密依赖。

## 目标与成功标准

目标是让用户能把当前账号的六类长期本地数据保存为一个带口令的版本化文件，并在数据仍可完整验证时恢复它。

成功标准：

- 一个备份包覆盖当前账号作用域内的本地原书、译本、词汇本、句子本、笔记和阅读器收藏。
- 创建备份前，六类数据全部经过现有权威解析器；任何一类损坏时停止创建。
- 使用浏览器 Web Crypto 实现 AES-256-GCM 与 PBKDF2-HMAC-SHA-256，不上传数据，不保存口令，不新增 npm 依赖。
- 备份包包含明确的格式版本、创建时间、来源账号作用域指纹、密钥派生参数、加密参数和密文。
- 恢复先完成零写入检查，向用户展示六类数量与创建时间，再要求明确确认。
- 执行恢复时整体替换六类数据；任一写入失败时立即尝试回滚，回滚失败绝不显示成功。
- 来源账号作用域与当前作用域不一致时拒绝恢复，避免把甲账号的本地数据误写到乙账号。
- 文件、解密负载、口令和派生参数均有固定预算，畸形输入不能触发无界内存或计算工作。
- 创建、检查、取消、失败、恢复成功和组件卸载后都清除 UI 中的口令与敏感状态，并尽力释放临时字节缓冲区和对象 URL。

## 非目标

第一版明确不实现：

- 云端备份、自动同步、定时备份或后台上传；
- 跨账号迁移、账号合并或从一个账号恢复到另一个账号；
- 与现有数据自动合并、逐项冲突解决或选择性恢复；
- 备份云端书籍、私有对象、数据库记录或服务端翻译任务；
- 备份 Session、Cookie、账号密码、恢复码、手机号、用户名或账号标签；
- 备份环境变量、模型密钥、平台 Secret、MCP 配置或付费资源状态；
- 备份上传草稿、云端导入标记、下载历史、临时对象 URL 或本地语音播放状态；
- 枚举、读取、展示或迁移其他账号及旧版未分区键；
- 忘记口令后的找回、平台托管密钥或绕过加密；
- 压缩、分卷、增量备份、二维码或移动端系统分享扩展。

## 方案比较与决定

### 方案 A：单一加密备份包，恢复时整体替换

六类数据经统一校验后进入一个版本化负载，再由用户口令加密。恢复先检查完整包，确认后以固定顺序替换六个当前账号作用域键。它的优点是边界清晰、可验证、易于说明和回滚，不会产生部分恢复后的混合状态；缺点是用户不能挑选分类或自动合并。

### 方案 B：每类数据单独导出和恢复

单个文件更小，也能按需恢复，但用户需要管理多份文件，原书与译本关系容易断裂，无法在一个认证边界内证明整组数据完整，恢复失败时也更容易留下跨分类不一致。

### 方案 C：明文 JSON 导出后由用户自行保管

实现简单，但原书正文、译文、笔记和收藏会以明文落盘，不符合默认保护隐私的产品原则，也不能可靠区分错误口令、文件篡改和截断。

采用方案 A。第一版宁可提供一个边界窄、可回滚的整体恢复流程，也不引入自动合并和冲突语义。

## 备份范围与权威来源

只读取当前页面已经确定的账号作用域，固定覆盖以下六个基础键：

| 分类 | 基础键常量 | 权威模块与解析器 |
| --- | --- | --- |
| 本地原书与完整章节 | `localLibraryBooksStorageKey` | `src/lib/library/local-library-storage.ts` 的 `parseStoredLocalLibraryBooksResult()` |
| 本地译本、章节译文和翻译进度 | `localTranslationsStorageKey` | `src/lib/library/local-translation-storage.ts` 的 `parseStoredLocalTranslationsResult()` |
| 词汇本 | `localVocabularyStorageKey` | `src/lib/study/local-study-storage.ts` 的 `parseStoredVocabularyItemsResult()` |
| 句子本 | `localSentencesStorageKey` | `src/lib/study/local-study-storage.ts` 的 `parseStoredSentenceItemsResult()` |
| 笔记 | `localNotesStorageKey` | `src/lib/study/local-study-storage.ts` 的 `parseStoredStudyNotesResult()` |
| 阅读器划词与划句收藏 | `localReaderSelectionsStorageKey` | `src/lib/reader/reader-selection-save.ts` 的 `parseReaderSelectionCollectionsResult()` |

`localUploadDraftStorageKey` 和 `cloudImportMarkerStorageKey` 明确排除。所有基础键必须通过 `buildScopedLocalStorageKey(baseKey, currentScope)` 解析为当前作用域的实际键；实现不得通过遍历 `localStorage.length`、`key()`、前缀搜索或其他方式发现键。

页面从现有 `data-local-storage-scope` 属性取得当前不可逆作用域值。缺失作用域时停止，显示现有“当前登录状态无法确定本地数据归属”语义；不得回退到无作用域键、其他账号键或旧版历史键。

## 文件格式合同

下载文件名固定为：

```text
stray-pages-backup-YYYY-MM-DD.spbackup
```

日期使用创建时本地日历日期，文件名只含 ASCII。MIME 类型使用 `application/octet-stream`，文件内容为 UTF-8 JSON。第一版外层封装固定为：

```ts
type LocalBackupEnvelopeV1 = {
  format: "stray-pages-browser-local-backup";
  version: 1;
  createdAt: string;
  sourceScopeFingerprint: string;
  kdf: {
    name: "PBKDF2";
    hash: "SHA-256";
    iterations: 600000;
    salt: string;
  };
  cipher: {
    name: "AES-GCM";
    keyLength: 256;
    tagLength: 128;
    iv: string;
  };
  ciphertext: string;
};
```

`salt`、`iv` 和 `ciphertext` 使用无换行的标准 Base64。导入时执行严格 Base64 校验，不接受空白、URL-safe 字母表、缺少必要填充、非规范重编码或尾随内容。盐解码后必须恰好 16 字节，IV 必须恰好 12 字节，密文必须至少包含 16 字节 GCM 标签。

版本 1 的外层对象、`kdf` 对象和 `cipher` 对象都只接受上面列出的精确属性集合；缺少属性或出现未知属性都拒绝。文件字节使用 `TextDecoder("utf-8", { fatal: true })` 解码，非法 UTF-8 不允许被替换字符静默修复。

`createdAt` 必须是可解析且重新序列化后相同的 UTC ISO 8601 字符串。`sourceScopeFingerprint` 直接使用现有不可逆账号作用域值，只用于恢复时精确相等比较；封装不包含用户 ID、用户名、手机号或账号标签。该字段不是跨账号迁移凭据。

版本 1 的导入允许范围是一个严格合同：KDF 只能是 PBKDF2-HMAC-SHA-256，迭代次数只能是 600,000，盐只能是 16 字节；加密只能是 256 位 AES-GCM、128 位标签和 12 字节 IV。任何算法、长度或参数变化必须通过新的外层 `version` 引入，版本 1 不做宽松猜测。

## 认证元数据与加密负载

外层元数据必须被 AES-GCM 认证，避免攻击者替换来源作用域、创建时间或派生参数。`additionalData` 是下列对象按所示属性顺序通过 `JSON.stringify()` 得到的 UTF-8 字节：

```ts
{
  format,
  version,
  createdAt,
  sourceScopeFingerprint,
  kdf: { name, hash, iterations, salt },
  cipher: { name, keyLength, tagLength, iv },
}
```

加密负载固定为：

```ts
type LocalBackupPayloadV1 = {
  schemaVersion: 1;
  data: {
    libraryBooks: StoredLocalLibraryBook[];
    translations: StoredLocalTranslation[];
    vocabulary: VocabularyStudyItem[];
    sentences: SentenceStudyItem[];
    notes: StudyNote[];
    readerSelections: {
      vocabularyTexts: string[];
      sentenceTexts: string[];
    };
  };
};
```

创建备份时不直接复制未经验证的原始 localStorage 字符串。每类原始值先经过现有结果解析器；解析成功后，把解析器返回的规范化记录放进负载。缺失键和有效空集合统一写成对应的空数组或空收藏对象。这样备份包自身始终是完整快照，不依赖“缺少字段代表保留当前数据”之类的隐含语义。

恢复检查时，负载六个字段必须全部存在且没有类型替代。实现将每类数据重新序列化后送回同一权威结果解析器，只有六类都返回成功才可生成预览。恢复写入时使用解析后的规范化数据，而不是未经验证的输入对象。

## 口令与 Web Crypto 合同

创建备份时提供“备份口令”和“再次输入”两个密码输入框。两次输入必须逐个 Unicode code point 完全一致，不执行大小写转换、去空格或 Unicode 归一化。口令长度为 12 至 128 个有效 Unicode code point，拒绝未配对代理项。界面明确提示：“请使用独立口令，不要复用账号密码；忘记口令后无法恢复备份。”

实现只使用：

- `crypto.getRandomValues()` 生成 16 字节盐和 12 字节 IV；
- `crypto.subtle.importKey()` 导入口令 UTF-8 字节；
- PBKDF2-HMAC-SHA-256 派生不可导出的 256 位 AES-GCM `CryptoKey`；
- `crypto.subtle.encrypt()` 和 `crypto.subtle.decrypt()` 完成认证加密。

不得使用 `Math.random()`、固定盐、固定 IV、自制加密、明文校验和或新加密库。口令不得写入 URL、localStorage、Cookie、日志、错误对象、分析事件或备份文件。组件不得提供显示口令、记住口令或自动填入账号密码的能力。

Web 平台不能保证 JavaScript 字符串立即从内存中抹除，因此产品不承诺物理级内存清零。实现必须做到可验证的最佳努力：操作结束或取消后清空 React 口令状态；临时 UTF-8、明文和密文字节数组在不再需要时调用 `fill(0)`；派生密钥保持不可导出并丢弃引用；不把明文负载和口令放进长期闭包、错误文本或调试日志。

## 资源预算与拒绝顺序

固定预算：

- 备份文件最大 16 MiB，即 `16 * 1024 * 1024` 字节；创建后最终 UTF-8 封装和导入文件都受此限制。
- 解码后的密文字节和解密后的 UTF-8 负载各自最大 12 MiB，即 `12 * 1024 * 1024` 字节。
- 口令最大 128 Unicode code point。
- PBKDF2 迭代次数固定 600,000，不接受更低或更高的版本 1 输入。
- 盐、IV、标签长度按文件格式合同固定，不允许由文件扩大。

创建时先计算规范化负载的 UTF-8 字节数，再加密并计算最终封装字节数；任何一步超限都停止下载。12 MiB 是明文和密文的独立安全上限，16 MiB 是含 Base64 与元数据的最终文件上限，因此接近 12 MiB 的负载仍可能因为最终封装超过 16 MiB 而被拒绝。

导入时按以下顺序限制工作：先检查扩展名，再检查 `File.size`，再读取字节、执行严格 UTF-8 解码并复核 16 MiB 文件预算，再解析外层固定字段，再严格解码 Base64 并检查密文字节预算，最后才执行 600,000 次 PBKDF2。不得在确认文件预算与固定 KDF 参数之前开始高成本派生。

## 数据完整性合同

现有单类解析器仍是字段形状、章节结构、译文段落对应关系和单类内部关系的权威来源。备份核心额外执行跨分类验证：

- 本地原书 ID 在原书集合中唯一；译本 ID 在译本集合中唯一；词汇、句子和笔记各自的记录 ID 唯一。
- 每个译本的 `originalBookId` 必须精确引用备份中的一个本地原书。
- 每个译本内部的任务、章节、段落数量与状态关系继续由 `isStoredLocalTranslation()` 的现有合同验证，不在备份模块复制另一套宽松规则。
- 阅读器收藏的词汇与句子分别按当前解析器验证字符串数组；第一版不自动去重或改变用户文本。
- 六类规范化数据重新序列化后的 UTF-8 总量必须处于 12 MiB 解密负载预算内。

任一条件失败都使整个备份不可创建或不可恢复。不得丢弃坏记录后继续、只恢复能解析的分类、把损坏数据替换为空集合或伪装成功。

## 架构与文件边界

### `src/lib/backup/local-backup-core.ts`

纯 TypeScript 核心，不访问 React、DOM、`window` 或 Web Crypto。职责：

- 定义版本 1 外层、认证元数据、负载、预览与稳定结果类型；
- 固定六类基础键、文件名、MIME 类型和字节预算；
- 把六个原始存储值交给现有解析器并构建规范化负载；
- 解析和严格验证外层结构、Base64、日期、版本与算法参数；
- 验证解密负载、ID 唯一性、原书与译本关系和预算；
- 生成不含正文的预览计数。

### `src/lib/backup/local-backup-crypto.ts`

只负责 Web Crypto 边界。通过小型运行时接口注入随机数和 `SubtleCrypto`，以便 Node 测试使用确定性替身。职责：

- 验证口令长度和两次输入一致性；
- 生成随机盐与 IV，派生不可导出密钥；
- 按固定认证元数据加密和解密；
- 严格控制临时字节缓冲区的生命周期；
- 把 Web Crypto 异常折叠为稳定错误，不返回底层异常、口令或密钥材料。

### `src/lib/backup/local-backup-restore.ts`

纯恢复事务协调器，通过注入的 `LocalStorageAdapter`、固定当前作用域和六类规范化数据工作。职责：

- 在任何写入前缓存六个当前作用域键的原始值；
- 把规范化分类序列化为待写入值；
- 按固定顺序替换六个键；
- 分类为空时调用删除而不是保留当前值；
- 写入失败时按相反顺序恢复操作前快照，并报告回滚结果。

该模块复用 `safeReadLocalStorage()`、`safeWriteLocalStorage()` 和 `safeRemoveLocalStorage()` 的稳定失败语义，但所有实际键只由固定基础键与已捕获作用域构造。

### `src/components/account/local-data-backup-panel.tsx`

客户端组件，负责文件、口令、预览、确认和状态 UI。组件读取当前作用域和六个固定键，调用纯核心与加密模块，再复用 `triggerBrowserDownload()` 下载。它不包含数据格式、加密算法或恢复事务的第二套实现。

### `src/app/me/page.tsx`

继续负责“我的”页服务端布局，在主要内容下方接入“本地数据备份”区域。该页面不读取 localStorage、不接触口令，也不把数据传到服务端。

## 创建备份流程

“创建备份”区域使用内联表单，不使用弹窗：

1. 用户输入备份口令和确认口令。
2. 组件捕获当前 `sourceScopeFingerprint`；缺失时停止。
3. 只读取由六个固定基础键和当前作用域构成的实际键。
4. 任一读取失败，显示现有本地存储错误语义并停止。
5. 六个原始值分别经过权威结果解析器；任一损坏时显示具体分类损坏，并停止。
6. 核心规范化数据，验证 ID、跨分类关系和 12 MiB 负载预算。
7. 加密模块生成随机盐与 IV，派生密钥，以认证元数据为 AAD 加密负载。
8. 核心构建外层 JSON，检查最终 16 MiB 文件预算。
9. 复用 `triggerBrowserDownload()` 创建 Blob、触发下载并在 `finally` 撤销对象 URL。
10. 无论成功或失败，都清空两个口令框、丢弃负载和密钥引用并清理临时字节数组。创建备份不修改六个现有键。

快速重复点击在一次创建过程中必须被禁用。用户得到“已准备下载”后只代表浏览器已收到下载请求，不宣称文件已被用户永久保存。

## 恢复检查流程：零写入阶段

“恢复备份”区域与创建区域分开，采用文件选择、口令和“检查备份”按钮。接受的文件扩展名只有 `.spbackup`，但扩展名不是信任依据。

检查阶段：

1. 检查文件名扩展名和 `File.size <= 16 MiB`。
2. 读取并解析外层，验证版本、算法、固定参数、编码和预算。
3. 捕获当前账号作用域；缺失时停止。
4. 使用用户口令、外层盐、IV 和固定 AAD 尝试认证解密。
5. 只有认证成功后，比较 `sourceScopeFingerprint` 与当前作用域；不一致时拒绝跨账号恢复。
6. 解析负载，六类数据全部重新走现有解析器，再执行唯一性、关系和预算检查。
7. 生成预览，展示备份创建时间、本地原书数、译本数、词汇数、句子数、笔记数和阅读器收藏数。阅读器收藏数等于词汇文本数与句子文本数之和，同时分别显示两项明细。

上述步骤不得调用 localStorage 的 `setItem()` 或 `removeItem()`。预览不得显示正文、译文、笔记内容、收藏文本、作用域指纹、盐、IV 或密文。

检查成功后，组件只在内存中保留已经验证并规范化的恢复候选和检查时作用域；立即清空口令输入。更换文件、再次检查、取消、作用域变化或组件卸载时必须使旧候选失效并清理引用。

## 恢复执行流程：整体替换阶段

预览下方固定显示未预选的确认框：“我了解恢复会替换当前账号的本地数据”。只有预览仍有效且用户勾选后，“恢复并替换”按钮才可用。

执行顺序：

1. 再次读取当前作用域，并与检查时作用域和备份来源作用域同时比较；任一变化都停止并丢弃候选。
2. 在任何修改前，读取并缓存六个实际键的原始字符串或缺失状态；任一读取失败则零写入退出。
3. 固定按本地原书、译本、词汇本、句子本、笔记、阅读器收藏的顺序执行。
4. 非空分类写入规范化 JSON；空分类删除当前对应键，确保备份中的空确实替换当前非空数据。
5. 每次变更后记录已尝试的键。任一写入或删除失败，立即停止后续写入，并按相反顺序把所有已尝试键恢复为步骤 2 的原始字符串或缺失状态；失败的当前键也纳入回滚，以防底层在抛错前发生部分变更。
6. 所有写入成功后才显示恢复成功，清除候选、快照、确认状态和敏感内存，并提示刷新页面以让所有工作区重新读取数据。

如果主写入失败但回滚全部成功，显示：“恢复失败，原有本地数据已恢复，未完成替换。”如果任何回滚动作失败，显示高优先级警告：“恢复失败，且无法完整还原原有本地数据。请不要继续编辑，并保留备份文件。”此状态使用 `role="alert"`，不得显示成功或“没有影响”。

同一窗口的 localStorage 修改不会自动向所有已挂载组件广播可靠更新，因此第一版成功后明确要求刷新，不伪装成所有页面已经热更新。恢复模块不自行调用强制刷新，避免用户看不到最终状态说明。

## 错误合同

UI 只使用稳定、可操作的中文错误，不暴露 DOMException 名称、堆栈、密钥、Base64 内容或底层存储键。

- 扩展名不符：“请选择 Stray Pages 备份文件（.spbackup）。”
- 文件超限：“备份文件超过 16 MiB，无法检查。”
- 不支持的明确版本或算法：“此备份版本当前无法恢复。”
- 错误口令、GCM 认证失败、密文篡改、密文截断、外层 JSON 截断或无法可靠区分的封装损坏，统一显示：“无法验证备份：口令错误或文件已损坏。”
- 认证成功但负载结构、关系或预算无效：“备份中的本地数据不完整或已损坏，未写入任何内容。”
- 账号作用域不一致：“此备份来自另一个账号，当前账号不能恢复。”
- 当前本地数据读取失败：“无法读取当前账号的本地数据，未开始恢复。”
- 主写入或回滚失败使用恢复执行流程中的固定文案。

解析流程可以在内部区分稳定错误码供测试断言，但不得把“口令一定错误”作为唯一解释，也不得提供认证预言式细节。

## UI 与可访问性

“本地数据备份”是“我的”页中的普通内联区域，不使用模态框、嵌套卡片堆叠或营销式大标题。它包含简短说明：“备份只在当前浏览器中加密，不会上传。恢复会替换当前账号的本地数据。”

布局要求：

- 桌面端创建和恢复可并列，窄屏自然堆叠；输入框、文件选择和按钮不横向溢出。
- 所有输入有可见 `label`，口令要求和不可找回说明通过 `aria-describedby` 关联。
- 进行中按钮禁用并使用明确动词，如“正在加密”“正在检查”“正在恢复”；不只用旋转图标表达状态。
- 普通成功提示使用 `aria-live="polite"`；认证失败、数据损坏、写入失败和回滚失败使用 `role="alert"`。
- 文件变化会清除旧预览和确认框，避免用户看到甲文件预览却执行乙文件。
- 预览计数使用语义化列表，不把颜色作为唯一状态表达。
- 普通用户页面使用“备份口令”“本地数据”“检查备份”等产品语言，不显示 PBKDF2、AES、KDF、Provider、token、API、审计或存储键。

## 隐私、费用与安全边界

- 所有读取限于六个固定的当前账号作用域键；不枚举 localStorage。
- 文件内容只在浏览器内解析、派生、加密、解密和下载；新模块不得包含 `fetch`、XHR、WebSocket、云 SDK、模型 SDK或 Blob 上传。
- 不创建、不试用、不绑定任何付费或需要付款方式的资源。
- 备份口令与账号密码完全独立，不进入服务端请求、日志、分析或持久化存储。
- AES-GCM 认证覆盖外层关键元数据和全部负载；任何不明确状态 fail closed。
- 只在认证和完整验证完成后产生恢复候选，只在明确确认后产生写入。
- 不新增 npm 依赖，不读取或修改真实 `.env`，不需要控制台登录或外部网站。

## 测试策略

实施严格执行 TDD，每一类生产行为先有失败测试。

### 备份核心单元测试

- 六类缺失键规范化为空数据；六类有效数据形成完整负载。
- 任一权威解析器返回损坏时整个创建失败，不产生部分负载。
- 原书、译本、词汇、句子和笔记 ID 重复被拒绝。
- 译本引用缺失原书被拒绝；有效关系通过。
- 预览七项展示值正确，阅读器收藏总数与两项明细一致。
- 日期、固定格式、版本、算法、参数和严格 Base64 验证。
- 16 MiB 文件、12 MiB 密文/明文边界的前一字节、等于边界和后一字节行为。
- 六个字段缺失、替代类型、额外宽松版本猜测和尾随内容被拒绝。
- AAD 属性顺序和 UTF-8 编码有固定测试向量。

### 加密单元与集成测试

- 12、128 code point 通过，11、129、未配对代理项和不一致确认被拒绝。
- 口令不被裁剪或归一化；不同 Unicode 序列产生不同输入字节。
- 盐 16 字节、IV 12 字节且每次由注入随机源重新生成。
- PBKDF2-SHA-256 固定 600,000 次，AES-GCM 256 位、128 位标签，密钥不可导出。
- 固定随机源下产生可复现封装，并能完整往返。
- 错口令、AAD 元数据变化、密文位翻转和截断折叠为同一认证失败码。
- 不支持版本和固定参数错误在高成本派生前被拒绝。
- 成功与失败路径都会执行临时缓冲区和引用清理。

### 恢复事务测试

- 检查阶段没有任何 `setItem()` 或 `removeItem()` 调用。
- 执行前六个快照读取完整；读取失败时零写入。
- 六类按固定顺序写入，备份空分类会删除当前非空键。
- 每个写入位置分别注入失败，后续键不写入，已尝试键按反序恢复。
- 原始缺失键在回滚时恢复为缺失，原始字符串逐字恢复。
- 失败键发生“先变更后抛错”时仍纳入回滚。
- 回滚全部成功与部分失败返回不同稳定结果，回滚失败绝不返回成功。
- 检查后账号作用域变化时拒绝执行且零写入。

### 组件合同

- “我的”页接入客户端面板，服务端页面不读取 localStorage 或口令。
- 创建、检查、预览、确认和执行状态的文案、禁用状态与无障碍属性正确。
- 两次创建点击不会并发；检查或恢复进行中不能重复提交。
- 更换文件、取消、失败、成功和卸载会清空预览、确认与口令状态。
- 只读取六个固定作用域键，不调用 `localStorage.key()` 或遍历长度。
- 创建路径复用 `triggerBrowserDownload()`，对象 URL 在成功和失败路径撤销。
- 新模块不包含网络、云服务、模型、付费资源或敏感日志代码。

### 回归与最终验证

- 现有本地导入、翻译、阅读、学习收藏、TXT/EPUB 下载和本地语音测试继续通过。
- `pnpm test`、`pnpm lint`、`pnpm typecheck`、`pnpm build`。
- `pnpm verify:zero-cost`。
- 凭据模式扫描、备份模块网络/云调用扫描和 `git diff --check`。
- 推送 `origin main`，核对本地与远端 SHA，等待 GitHub Actions 达到 `completed/success`。

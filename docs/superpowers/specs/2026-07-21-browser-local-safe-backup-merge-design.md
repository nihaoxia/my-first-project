# 浏览器本地备份安全合并设计

日期：2026-07-21

状态：方案已批准（依据用户对产品与技术决策的持续授权）

依赖规格：

- `docs/superpowers/specs/2026-07-21-browser-local-data-backup-design.md`
- `docs/superpowers/specs/2026-07-21-browser-local-selective-restore-design.md`

## 1. 背景

Stray Pages 已支持把当前账号作用域内的六类浏览器长期数据加密为版本 1 `.spbackup` 文件，并在检查、同账号验证、五组范围选择和明确确认后整体替换所选分类。当前选择性恢复适合把某个分类完整恢复到备份状态，但用户只想找回误删记录时，整体替换仍会覆盖备份创建以后新增或修改的当前记录。

正式未实现范围仍包含“备份记录的自动合并”。本阶段在不改变备份文件、加密参数、同账号限制和零上传边界的前提下，增加安全补回式合并：保留当前记录，只把备份中当前缺失的记录补回；无法无损自动判断的冲突不覆盖当前数据。

## 2. 目标

- 在恢复页面提供“安全合并”和“替换所选分类”两种恢复方式。
- 安全合并默认保留当前记录，并补回备份中缺失的记录。
- 合并前展示只含数量的零写入预览。
- 同 ID 冲突采用确定且不会静默覆盖当前内容的规则。
- 笔记的本地递增 ID 冲突时重新分配 ID，两份笔记都保留。
- 阅读器收藏按规范化文本做稳定并集。
- 原书与译本继续作为不可拆分的 `library` 组，并在合并后保持引用完整。
- 执行时验证预览使用的当前快照仍然有效；当前数据变化时零写入并要求重新预览。
- 只读取、写入和回滚选中范围内的键；没有变化的选中键不写入。
- 保留现有替换事务、反向回滚、零上传、零新增依赖和零收费资源边界。

## 3. 非目标

本阶段不实现：

- 备份记录覆盖当前同 ID 记录；
- 按时间戳自动判断“较新版本”；
- 逐记录手工冲突选择或正文差异对比；
- 合并删除标记、推断用户曾删除的记录或同步删除操作；
- 跨账号迁移或跨账号恢复；
- 云端同步、云端自动备份、定时备份或跨设备合并；
- 修改 `.spbackup` 格式、版本、PBKDF2、AES-GCM、AAD、口令规则或大小预算；
- 在备份文件中保存恢复方式、选择范围、预览或冲突结果；
- 读取上传草稿、云端导入标记或六类范围之外的浏览器数据；
- 新增 npm 依赖、网络请求、模型调用、日志上报或云资源。

“安全合并”只表示把备份中缺失的数据补到当前浏览器，不表示双向同步，也不表示备份内容一定比当前内容新。

## 4. 方案选择

### 4.1 采用方案 A：安全补回式合并

当前浏览器记录优先。备份独有记录按备份顺序追加；同 ID 且内容不同的记录不覆盖当前版本，并在预览中计为冲突。笔记因使用本地递增 ID，发生同 ID、不同内容冲突时给备份笔记分配新 ID，两份都保留。

该方案无需不可靠的时间比较，不会静默覆盖用户在备份以后继续修改的数据，能够直接解决“找回误删内容”的主要需求。

### 4.2 未采用方案 B：较新时间戳优先

原书有 `savedAt`，译本有 `updatedAt`，但词汇、句子和阅读器收藏没有时间戳；笔记的 `updatedAt` 当前可以是“刚刚”等展示文本。按时间合并会在不同分类采用不同规则，并把不可比较的时间错误解释为版本顺序，因此不采用。

### 4.3 未采用方案 C：逐记录冲突解决器

逐条展示当前版本与备份版本最灵活，但需要正文截断、敏感内容展示、分页、批量选择和复杂确认状态。本阶段先建立确定、安全的自动合并底座，不增加记录级冲突界面。

## 5. 恢复方式与页面流程

新增固定恢复方式：

```ts
type LocalBackupRestoreMode = "merge" | "replace";
```

备份检查成功后：

1. 默认恢复方式为 `merge`；
2. 五个恢复组默认全选；
3. 页面继续展示备份创建时间和六类备份计数；
4. 页面显示“恢复方式”原生单选组；
5. 安全合并模式显示“预览合并结果”按钮；
6. 替换模式沿用现有确认框和“恢复所选数据”按钮。

安全合并流程：

1. 用户选择恢复组；
2. 点击“预览合并结果”；
3. 页面只读取选中组映射出的当前作用域键；
4. 当前数据严格解析成功后，生成合并预览和内存快照；
5. 页面按分类展示数量，不展示记录正文、ID、文件名、指纹或冲突内容；
6. 只有预览包含至少一个实际新增或重分配记录时，才允许确认；
7. 用户确认后点击“合并所选数据”；
8. 执行阶段重新读取选中键，并与预览时快照逐字符串比较；
9. 任一选中键发生变化时返回 `CURRENT_DATA_CHANGED`，零写入并要求重新预览；
10. 快照未变化时，用同一纯合并函数重新计算计划并执行事务。

以下变化都会立即撤销确认并清除旧合并预览和内存快照：

- 恢复方式变化；
- 任一恢复组变化；
- 选择另一个文件；
- 再次检查文件；
- 取消恢复；
- 恢复或合并执行结束；
- 当前账号作用域变化；
- 页面卸载。

## 6. 固定合并规则

### 6.1 通用 ID 记录规则

原书、译本、词汇和句子按 `id` 合并：

- 当前数据中的记录保持原顺序；
- 备份中当前没有的 ID 按备份原顺序追加；
- 同 ID 且规范化内容相同：保留当前记录，计为“已存在”；
- 同 ID 但规范化内容不同：保留当前记录，计为“冲突保留当前”；
- 不修改、拼接或部分覆盖同 ID 当前记录；
- 当前或备份分类内部出现重复 ID 时停止合并。

内容比较使用浏览器可执行的稳定 JSON 规范化：对象键递归排序，数组顺序保留，`undefined` 属性按标准 JSON 语义忽略。比较只用于预览分类，不改变写回的数据对象。

### 6.2 原书与译本

`library` 继续同时覆盖本地原书和本地译本，不能拆选：

1. 先合并原书；
2. 再合并译本；
3. 验证最终每个译本的 `originalBookId` 都存在于最终原书集合；
4. 引用缺失时返回 `MISSING_ORIGINAL_BOOK`，零写入；
5. 原书没有变化而译本有新增时，只写译本键；
6. 译本没有变化而原书有新增时，只写原书键。

同 ID 当前原书即使与备份内容不同也优先保留。只要最终原书 ID 存在，备份独有译本可以补回；本阶段不比较同 ID 原书正文是否与译本来源完全相同。

### 6.3 词汇和句子

词汇与句子只按现有稳定 `id` 合并：

- 不额外按单词、句子正文、书名或章节标题推断重复；
- 不拼接 `note`、`explanation` 或其他字段；
- 同 ID 当前记录优先；
- 备份独有 ID 按备份顺序追加。

这避免把来自不同书籍、章节或上下文的相同文本错误合并为一条记录。

### 6.4 笔记

笔记使用以下规则：

- 备份笔记 ID 当前不存在：按备份顺序追加；
- 同 ID 且规范化内容相同：保留当前笔记，计为“已存在”；
- 同 ID 但内容不同：保留当前笔记，并给备份笔记分配新的 `note-local-N` ID 后追加；
- 新编号从当前和备份所有 `note-local-N` 中的最大数字加一开始；
- 每次分配后检查所有已使用 ID，若碰撞则继续递增；
- 非 `note-local-N` 的既有 ID 也进入已使用集合，但不参与最大数字计算；
- 重分配只改变 `id`，不改标题、来源、更新时间或正文；
- 多个冲突按备份原顺序连续分配，结果确定。

笔记没有跨分类引用，因此重新分配 ID 不需要修改其他记录。

### 6.5 阅读器收藏

`vocabularyTexts` 和 `sentenceTexts` 分别合并：

- 比较键为 `text.trim().toLowerCase()`；
- 当前数组保持原顺序和原始大小写；
- 当前数组内部已有的重复项也保持原样，但全部进入已存在比较集合；安全合并不能借去重之名删除当前条目；
- 备份文本规范化后已存在则计为“已存在”；
- 备份独有文本使用 `trim()` 后的原始大小写，按备份顺序追加；
- 空白文本在当前或备份中出现时视为当前数据异常，停止合并；
- 词汇文本和句子文本相互独立，不跨集合去重。

## 7. 合并计划与预览模型

新增纯逻辑模块：

```text
src/lib/backup/local-backup-merge.ts
```

职责：

- 接收已选组对应的当前原始值；
- 接收已解密和严格验证的版本 1 备份 payload；
- 独立解析每个选中分类，不要求读取未选中键；
- 生成选中数据键的稳定合并结果；
- 生成每个恢复组的预览统计；
- 标记实际发生变化的数据键；
- 验证 ID 唯一性、译本引用和合并大小预算；
- 不访问 `window`、`localStorage`、网络、Cookie、URL 或日志。

每组预览统计固定为：

```ts
type LocalBackupMergeGroupPreview = {
  current: number;
  backup: number;
  added: number;
  existing: number;
  conflictsKeptCurrent: number;
  rekeyed: number;
};
```

其中 `rekeyed` 只可能在笔记组非零。阅读器收藏的数量为两类文本数量之和；页面仍可在辅助说明中分别展示词汇文本和句子文本计数。

合并检查结果只存在当前页面内存：

```ts
type LocalBackupMergeInspection = {
  selectedGroups: readonly LocalBackupRestoreGroup[];
  inspectedScopeFingerprint: string;
  currentRawValues: Partial<Record<LocalBackupDataKey, string | null>>;
  preview: Readonly<Partial<Record<LocalBackupRestoreGroup, LocalBackupMergeGroupPreview>>>;
  changedDataKeys: readonly LocalBackupDataKey[];
};
```

`currentRawValues` 只包含选中组对应的键。候选失效时必须和解密候选一起清除，不持久化、不上传、不写入日志。

## 8. 读取、预算与事务

### 8.1 只读预览

恢复编排模块提供只读检查函数。它按以下顺序执行：

1. 验证来源作用域、检查作用域和当前作用域一致；
2. 验证选择非空、无重复、无未知组；
3. 按六键权威顺序展开选中目标；
4. 只读取选中目标的当前原始值；
5. 任一读取失败时返回 `READ_FAILED`；
6. 调用纯合并模块；
7. 返回预览、变化键和内存快照；
8. 全过程零写入。

### 8.2 大小预算

对所有实际变化键的序列化目标值进行 UTF-8 计数：

- 字节数总和必须为正且不超过现有 `localBackupPayloadByteLimit`；
- 没有变化时返回成功预览，但 `changedDataKeys` 为空，执行按钮禁用；
- 超过预算时返回 `MERGED_DATA_TOO_LARGE`，零写入；
- 不为计算预算读取未选中分类；
- 浏览器实际存储配额仍由安全写入结果处理，写入失败进入现有回滚事务。

### 8.3 最终执行

`restoreLocalBackup()` 增加必填 `mode`。替换模式保持当前行为；合并模式还必须传入同一候选产生的 `mergeInspection`。

合并执行顺序：

1. 作用域验证；
2. 恢复方式验证；
3. 选择验证；
4. 合并检查对象结构、作用域和选择一致性验证；
5. 只读取选中键的最新快照；
6. 与 `mergeInspection.currentRawValues` 逐键严格比较；
7. 不一致时返回 `CURRENT_DATA_CHANGED`，零写入；
8. 使用最新快照重新生成合并计划；
9. 预览统计、变化键或目标序列化值与检查结果不一致时返回 `CURRENT_DATA_CHANGED`；
10. 没有变化时返回 `{ ok: true, outcome: "unchanged" }`；
11. 只把实际变化键加入主写入目标；
12. 按六键权威顺序写入；
13. 在主修改前把当前键加入 `attempted`，因此失败键也进入回滚；
14. 失败后按相反顺序回滚全部已尝试变化键；
15. 一个回滚失败后继续其余回滚；
16. 返回合并成功、完整回滚或回滚失败结果。

未选中键在预览、快照比较、解析、预算、写入和回滚中始终零接触。选中但未变化的键可以读取和解析，但不写入或回滚。

## 9. 结果与错误文案

安全合并模式确认文案：

```text
我了解安全合并会保留当前记录，并补回备份中缺失的记录；同 ID 冲突默认保留当前版本。
```

按钮文案：

- 预览：`预览合并结果`
- 执行：`合并所选数据`
- 执行中：`正在合并`

稳定结果文案：

- 合并成功：`所选备份数据已安全合并，当前记录未被覆盖。请刷新页面，让相关工作区重新读取数据。`
- 无变化：`当前数据已包含所选备份记录，无需写入。`
- 当前数据变化：`预览后当前数据发生了变化，未写入任何内容。请重新预览。`
- 当前分类异常：`当前所选分类的数据异常，无法安全合并，未写入任何内容。`
- 结果过大：`合并后的所选数据超过本地安全预算，未写入任何内容。`
- 完整回滚：`合并失败，所选分类的原有本地数据已恢复，未完成写入。`
- 回滚失败：`合并失败，且无法完整还原所选分类的原有本地数据。请不要继续编辑，并保留备份文件。`

替换模式继续使用现有“所选分类整体替换”文案。普通用户文案不出现底层键名、快照、事务、KDF、PBKDF2、AES、AAD、Provider、token、API 或审计术语。

## 10. 可访问性与布局

- 恢复方式使用带可见标题的 `<fieldset>`、`<legend>` 和原生单选框；
- 当前选中方式通过原生 `checked` 状态表达，不只依赖颜色；
- 合并预览使用标题、简短说明和每组可读数量，不使用横向宽表；
- 冲突统计说明“保留当前版本”，不使用仅开发者理解的术语；
- 无新增内容和错误提示分别使用 `role="status"`、`role="alert"`；
- 键盘可以完成方式选择、范围选择、预览、确认、执行和取消；
- 窄屏保持单列，桌面可两列；不得产生横向滚动；
- 沿用现有内联卡片，不新增弹窗、向导、动画或新设计系统。

## 11. 安全、隐私与费用边界

- 不修改现有备份文件解析和 Web Crypto 实现；
- 不改变版本 1 文件格式、KDF、加密、AAD、口令或账号指纹；
- 不把当前数据、备份数据、快照、选择、预览、冲突或结果写入文件、Cookie、URL、日志或分析服务；
- 不枚举 `localStorage`；只使用选中组映射出的固定作用域键；
- 不读取未选中键，即使只用于验证或预算也不允许；
- 不发送网络请求，不调用 EdgeOne、Blob、KV、Models、COS、TTS 或其他云服务；
- 不新增依赖，不创建收费资源，不改变零费用门禁；
- 页面取消、文件变化、再次检查、执行结束和卸载后不保留合并快照。

## 12. 测试策略

### 12.1 纯合并逻辑

- 当前顺序保持，备份独有 ID 按备份顺序追加；
- 相同 ID、相同内容计为已存在；
- 相同 ID、不同内容保留当前并计为冲突；
- 原书与译本先后合并并保持引用完整；
- 词汇和句子不按正文误合并；
- 笔记相同内容去重，不同内容冲突稳定重分配 ID；
- 多个笔记冲突连续分配且避开所有当前和备份 ID；
- 阅读器收藏 trim、忽略大小写去重，保持当前写法和顺序；
- 当前空白收藏文本、重复 ID、畸形分类或缺失原书稳定拒绝；
- 变化键顺序固定；
- 无变化计划不产生写入目标；
- 合并预算边界值通过，超一字节拒绝。

### 12.2 预览与事务

- 作用域校验先于方式、选择和存储访问；
- 非法方式、选择和检查对象零存储访问；
- 预览只读取选中键并且零写入；
- 每个选中快照读取位置失败时零写入；
- 预览后任一选中键变化时零写入；
- 未选中键变化不影响执行，因为它们从未读取；
- 执行重新计算计划并与预览一致；
- 只写入实际变化键；
- 每个主写入失败位置反向回滚已尝试变化键；
- 失败键进入回滚；
- 一个回滚失败后继续其余回滚；
- 替换模式现有回归全部保持。

### 12.3 UI 合同与浏览器验收

- 检查成功后默认安全合并并默认全选五组；
- 恢复方式、选择或文件变化清除预览和确认；
- 合并模式没有有效预览时不能确认或执行；
- 无新增记录时显示稳定提示并禁用执行；
- 预览只显示数量，不显示用户正文和底层 ID；
- 确认和按钮文案随方式准确变化；
